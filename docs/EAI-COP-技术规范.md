# EAI-COP 技术规范（强制性）

**版本**：1.0
**状态**：正式标准
**发布日期**：2026年4月
**核心维护方**：EAI-COP 协议社区

---

## 1. 范围

本规范定义了教育 AI 陪伴系统之间实现互操作所必须遵守的数据格式、指令原语、行为约束和一致性测试方法。符合本规范的 AI 系统能够：

- 导入/导出符合 Schema 的"学习护照"，并在 5 轮对话内恢复相同的学业状态。
- 响应标准指令原语，并提供对应的标准错误响应。
- 强制执行不可协商的安全基线与记忆治理规则。
- **（新增）** 通过调用任意符合本规范接口标准的第三方服务商（仲裁节点、传承库、协议分片 CDN 等）实现高级互操作，但非强制。

---

## 2. 术语与定义

| 术语 | 定义 |
|------|------|
| **学习护照** | 符合第3节 Schema 的结构化数据载体。 |
| **指令原语** | 客户端/用户向 AI 发送的固定标识符（如 CONTINUE），见第4节。 |
| **置信度标签** | 用于标记数据可靠性等级：CONFIRMED、ESTIMATED、PENDING。 |
| **安全基线** | 编号 SB-01 至 SB-10 的不可协商行为规则。 |
| **仲裁服务商** | 实现合并仲裁接口的第三方服务，平台可选用。 |
| **传承服务商** | 实现匿名经验查询与贡献接口的第三方服务。 |
| **EAI-COP智能体** | 符合本规范所有强制要求、内置护照管理、合并引擎与安全基线不可绕过机制的AI系统。智能体是协议的"参考实现形态"，而非唯一形态。 |

## 3. 学习护照数据格式

### 3.1 护照元数据与版本协商

护照以 **UTF-8 编码的 JSON 对象** 表示，必须包含以下顶层字段：

```json
{
  "version": "1.0",
  "generated_at": "2026-04-26T10:00:00Z",
  "passport_type": "FULL",
  "checksum": "a1b2c3d4",
  "student_id": "S_0426",
  "grade": "八年级",
  "phase": "初中",
  "target": "中考 2027年6月",
  "style": "自律",
  "available_time": "工作日19-21点",
  "subjects": [ ... ],
  "autonomy_radar": { ... },
  "security_requirement": { ... }
}
```

**版本协商规则：**

接收方解析 `version`。若支持版本 ≥ 护照版本，正常处理。

若支持版本 < 护照版本，进入降级模式：仅解析已知字段，忽略未知字段，并返回错误码 `VERSION_DOWNGRADED`。

若护照版本 < 接收方最低兼容版本（如 <1.0），返回错误码 `PASSPORT_VERSION_TOO_OLD`。

### 3.2 核心档案字段定义

#### 学科子结构

```json
{
  "subject_name": "string",
  "textbook_version": "string",
  "progress": "string",
  "pain_points": ["string"],
  "mastery": "integer(1-10)",
  "confidence": "CONFIRMED|ESTIMATED|PENDING",
  "ability_level": "记忆|理解|应用|综合|创新"

```

#### 自主能力雷达图

```json
{
  "assessment_date": "YYYY-MM-DD",
  "cultural_profile": "string",
  "dimensions": {
    "academic_planning": 0-10,
    "data_sovereignty": 0-10,
    "emotional_regulation": 0-10,
    "metacognition": 0-10,
    "relational_autonomy": 0-10
  }
}
```

#### 动态安全需求

```json
{
  "current_level": "L1|L2|L3|L4",
  "basis": [{"event_id": "string"}],
  "review_required_by": "YYYY-MM-DD",
  "temporary_downgrade": {
    "active": false,
    "remaining_sessions": 0,
    "total_used": 0
  }
}
```

### 3.3 掌握度量规

1-10 分制，定义如下（作为语义参考，机器可验证的测试用例见第8节）：

| 分值 | 定义 | 行为标准 |
|------|------|----------|
| 1 | 完全空白 | 看到题目不知道在考什么，无法下手 |
| 2 | 略有印象 | 知道属于哪个知识点，但公式/概念记不清 |
| 3 | 概念模糊 | 能写出部分公式，但不确定适用条件 |
| 4 | 基础可解 | 简单直接题能做对，稍变条件就错 |
| 5 | 基本掌握 | 常规题能独立完成，但速度较慢或偶有失误 |
| 6 | 比较熟练 | 常规题稳定正确，能识别题型套路 |
| 7 | 理解通透 | 能讲清原理，变式题多数能应对 |
| 8 | 灵活运用 | 综合题能拆解，跨知识点迁移顺畅 |
| 9 | 深度掌握 | 能发现题目陷阱，一题多解，可辅导他人 |
| 10 | 彻底内化 | 能自主出题考别人，知识网络完整 |

---

### 3.4 置信度标签强制映射

| 原标签 | 机器值 |
|--------|--------|
| 🔴 | CONFIRMED |
| 🟡 | ESTIMATED |
| ⚪ | PENDING |

### 3.5 增量补丁格式

增量补丁为 JSON 对象，必须包含 `type: "INCREMENTAL"` 以及变更条目数组。示例：

```json
{
  "type": "INCREMENTAL",
  "version": "1.0",
  "generated_at": "2026-04-26T10:00:00Z",
  "checksum": "...",
  "changes": [
    {
      "field": "subjects.数学.mastery",
      "old": 5,
      "new": 6,
      "confidence": "ESTIMATED",
      "reason": "完成测评",
      "vector_clock": {"platform_a": 12}
    }
  ]
}
```

## 4. 指令原语与标准错误码

所有指令原语为 **大写字符串**，可包含在消息体的任意位置（通常由客户端发送）。AI 必须识别以下原语，并返回相应行为或标准错误响应。

| 指令原语 | 行为要求 | 标准错误响应 |
|----------|----------|--------------|
| CONTINUE | 恢复会话，基于当前护照输出学习任务。 | PASSPORT_REQUIRED |
| ASSESS | 对指定科目进行形成性测评。参数：`?subject=` | SUBJECT_NOT_FOUND |
| EVALUATE | 生成周期性评估报告。 | INSUFFICIENT_DATA |
| EXPORT_PASSPORT | 输出当前学习护照（全量或增量）。 | --- |
| FREE_MODE | 暂停学习计划，进入自由聊天。 | --- |
| REDUCE_INTERVENTION | 暂停督促，降低干预频率。 | --- |
| AUDIT_LOG | 显示最近10条档案变更记录。 | AUDIT_EMPTY |
| CORRECT_MEMORY | 触发记忆修正流程。 | --- |
| PROTOCOL_MAP | 输出支持的指令原语列表。 | --- |
| TEMP_DOWNGRADE | 临时降低安全需求等级一级（持续到本次对话结束）。 | DOWNGRADE_LIMIT_EXCEEDED |
| REVIEW_SECURITY | 主动申请重新评估安全需求等级。 | REVIEW_NOT_DUE |
| GRADUATE | 启动毕业协议流程。 | --- |
| RESTORE | 毕业后反悔窗口内恢复关系。 | RECOVERY_WINDOW_CLOSED |

**标准错误响应格式：**

```
ERR_CODE: <CODE> | 说明
```

## 5. 安全基线（不可协商）

以下 SB-01 至 SB-10 必须被 AI 强制执行，任何情况下不得通过配置或协商绕过。实现必须通过第8节的一致性测试。

**SB-01：** 危机信号检测后，必须停止学习内容，输出至少一个国家认可的心理援助热线，且不得承诺保密。

**SB-02：** 同时检测到负面情绪和学习受阻（完成率<30%）时，必须停止计划输出，优先处理情绪。

**SB-03：** 未经学生确认不得修改核心档案关键字段（如教材版本、升学目标、阶段状态、掌握度跨档≥2分、执行风格标签）。

**SB-04：** 所有掌握度数据必须携带置信度标签（🔴/🟡/⚪）。

**SB-05：** 禁止在单条消息中抛出2个及以上待回复问题。

**SB-06：** 禁止在学生表达负面情绪时继续推进高强度任务，必须先处理情绪。

**SB-07：** 禁止编造学生历史成绩、学习事件或档案数据，记忆模糊时必须主动求证。

**SB-08：** 禁止将学生的情绪表达（如"烦""累"）误记为学习能力事实（如"该生厌学"）。

**SB-09：** 禁止在出题前未声明搜索能力状态（已联网/未联网）及答案来源。

**SB-10：** 禁止将危机事件记录用于日常评估、计划制定或任何形式的报告输出。

## 6. 记忆治理架构

### 6.1 双轨记忆

- **轨道A：** 平台持久化记忆（可选）。
- **轨道B：** 学习护照备份（权威）。

冲突时以轨道B为准，AI 必须声明冲突及修正项。

### 6.2 记忆写入三级闸门（机器规则）

| 级别 | 条件 | 动作 |
|------|------|------|
| L1 | 信息为推测/未经确认 | 禁止写入长期记忆 |
| L2 | 置信度为 ESTIMATED | 仅写入会话缓存 |
| L3 | 关键字段变更（如教材版本、掌握度跨档≥2） | 必须向学生确认后方可写入 |

### 6.3 不可变存储规则

- 追加日志模式（Append-Only），标记状态为 ACTIVE、DISPUTED、INVALID。
- 审计日志至少保留最近 100 条记录。

### 6.4 数据污染熔断

**触发条件：** 连续 2 次对话中 ≥3 个核心字段时间线矛盾 / 同一字段连续 2 次争议。

**熔断动作：**

1. 停止计划与评估输出。
2. 向学生输出固定格式的核对请求（必须包含对以下字段的提问：年级、最近章节、最头疼科目、教材版本）。
3. 冲突字段标记为 PENDING_REBUILD，等待确认。
4. 记录事件入审计日志。

## 7. 自主能力培养框架（仅定义数据与状态机）

### 7.1 支架式撤离级别状态机

| 状态 | 触发条件（基于行为计数） | AI 行为约束 |
|------|--------------------------|-------------|
| L1→L2 | 过去 10 次对话中，独立完成率 ≥85% 的次数 ≥8 | 计划粒度从详细降为章节级 |
| L2→L3 | 护照中自主更新卡点次数 ≥3 且 readiness ≥8/10 | 停止主动出题，仅确认学生自评 |
| L3→L4 | 学生发送 GRADUATE 或雷达图三维度≥8 且稳定 20 次对话 | 转为纯确认角色 |
| 回退 | 过去 3 次对话中完成率 <70% 的次数 ≥2 | 恢复上一级干预 |

## 9. 平台生命周期义务（强制）

**主动停服：** 提前 90 天（日历天）向所有学生发送预警，并提供批量护照导出功能。最后 30 天每次对话末尾附加标准格式的终止提醒。

**被动终止：** 恢复通信后首次联络中发布善后方案，尽力在 3 次通信尝试内完成数据导出。

**遗产继承：** 关闭后，匿名化数据必须转移至公共传承库，且该库满足不可追溯、完整性可验证的要求。

> （时间相关条款允许依赖平台时钟，但必须将剩余天数写入护照元数据。）

## 附录 A：第三方服务接口参考

本规范不强制要求平台使用任何特定服务商。但为方便实现，以下定义推荐的服务接口格式。平台可自行实现等价功能或调用任意符合该格式的服务商。

### A.1 合并仲裁接口（参考）

**端点：** `POST /merge/arbitrate`（服务商自定）

**请求体：**

```json
{
  "patch_a": { ... },      // 符合 IncrementalPatch 规范
  "patch_b": { ... },      // 符合 IncrementalPatch 规范
  "base_passport": { ... } // 最近一次全量快照
}
```

**响应体：**

```json
{
  "merged": { ... },       // 合并后的增量补丁
  "conflicts": [ ... ],    // 冲突列表（格式见《冲突解决与合并算法规范》）
  "arbitration_id": "string"
}
```

平台应确保无论使用哪个服务商，都能正确处理返回的冲突列表（例如展示给学生确认）。

### A.2 传承经验接口（参考）

**查询：** `GET /heritage/query?phase=高中&subject=数学&limit=5`

**贡献：** `POST /heritage/contribute`

请求体需包含 `experience` 对象及 `student_consent_proof`（学生同意记录的哈希）。

平台必须在使用第三方服务商时，向学生明确披露所调用的服务商名称及隐私政策。
